تطبيق تحديد الإحداثيات هو أداة متقدمة لرسم الإحداثيات الجغرافية وحساب المساحات. مصمم خصيصاً للاستخدام الزراعي والمساحي مع واجهة عربية سهلة الاستخدام.

رسم الخرائط التفاعلية

  • خرائط Google عالية الدقة
  • أنواع خرائط متعددة (طرق، أقمار صناعية، هجين، تضاريس)
  • عرض الإحداثيات في الوقت الفعلي
  • تحويل تلقائي بين أنظمة الإحداثيات
  • دعم أنظمة إحداثيات متعددة (UTM و Lat/Lon)

إدارة المضلعات المتعددة

  • رسم عدة مضلعات في نفس الوقت
  • إدارة منفصلة لكل مضلع
  • ألوان مختلفة لكل منطقة
  • حسابات منفصلة لكل مضلع
  • تصدير جميع المضلعات معاً

حسابات متقدمة

  • حساب المساحة بالدونم
  • قياس المحيط بالأمتار
  • حساب المسافات بين النقاط
  • نتائج فورية ودقيقة
  • إجمالي المساحات لجميع المضلعات

تصدير البيانات

  • تصدير JSON: يحتوي على جميع البيانات بتنسيق منظم
  • تصدير CSV: ملف جدول بيانات منظم للتحليل
  • تصدير KML: ملف خرائط Google Earth
  • تصدير متعدد: تصدير جميع المضلعات في ملف واحد
  • تسمية تلقائية: يتم تسمية الملفات بالتاريخ والوقت

كيفية استخدام التطبيق

إدخال الإحداثيات

الخطوة الأولى: اختر نظام الإحداثيات (UTM أو Lat/Lon)
الخطوة الثانية: انتقل إلى قسم "إدخال الإحداثيات"
الخطوة الثالثة: أدخل الإحداثيات بالتنسيق المطلوب
الخطوة الرابعة: اضبط خيارات الرسم (اللون، الشفافية، اسم المالك)
الخطوة الخامسة: اضغط "رسم المضلع" لعرض النتائج
الخطوة السادسة: كرر العملية لإضافة مضلعات أخرى

رسم الخطوط

الخطوة الأولى: انقر بالزر الأيمن للماوس في نقطة البداية
الخطوة الثانية: انقر بالزر الأيمن مرة أخرى في نقطة النهاية
الخطوة الثالثة: ستظهر المسافة على الخط تلقائياً

إدارة المضلعات

عرض المضلع: انقر على "عرض" في قائمة المضلعات
حذف المضلع: انقر على "حذف" لإزالة مضلع معين
تحديد المضلع: انقر على المضلع في الخريطة لتحديده

تنسيق الإحداثيات

أنظمة الإحداثيات المدعومة

يدعم التطبيق نظامين للإحداثيات:

نظام UTM

  • تنسيق: Easting,Northing
  • مثال: 123456.78,9876543.21
  • منطقة UTM: 38N
  • مناسب للمسح الزراعي

نظام Lat/Lon

  • تنسيق: Latitude,Longitude
  • مثال: 31.9539,35.9106
  • نظام WGS84
  • مناسب للاستخدام العام

التنسيقات المدعومة

يقبل التطبيق الإحداثيات بالتنسيقات التالية:

# تنسيق نقطة واحدة Easting,Northing (UTM) 123456.78,9876543.21 Latitude,Longitude (Lat/Lon) 31.9539,35.9106 # تنسيق نقاط متعددة (سطر واحد لكل نقطة) 123456.78,9876543.21 234567.89,9876544.32 345678.90,9876545.43 # تنسيق نقاط متعددة مع فاصل * 123456.78,9876543.21*234567.89,9876544.32 345678.90,9876545.43*456789.01,9876546.54

ملاحظات مهمة

  • اختر نظام الإحداثيات المناسب قبل إدخال البيانات
  • الحد الأدنى للنقاط هو 3 نقاط لرسم المضلع
  • استخدم الفاصلة (,) للفصل بين الإحداثيات
  • استخدم * للفصل بين النقاط في نفس السطر
  • يمكن إدخال عدة أسطر للنقاط المتعددة
  • سيتم تحويل الإحداثيات تلقائياً للعرض على الخريطة

خيارات التخصيص

تخصيص المظهر

  • لون المضلع: اختر أي لون من لوحة الألوان
  • الشفافية: اضبط مستوى الشفافية من 10% إلى 100%
  • اسم المالك: أدخل اسم الفلاح أو المالك للعرض على الخريطة
  • نوع الخريطة: اختر بين الطرق، الأقمار الصناعية، الهجين، أو التضاريس

التحكم في الخريطة

  • التكبير والتصغير: استخدم عجلة الماوس أو أزرار + و -
  • التحريك: اسحب الخريطة بالماوس للتنقل
  • عرض الإحداثيات: انقر بالزر الأيمن في أي مكان على الخريطة
  • تغيير نوع الخريطة: استخدم أزرار الراديو في قسم التحكم

التراجع والإعادة

  • التراجع: استخدم زر "تراجع" لإلغاء الإجراء الأخير
  • الإعادة: استخدم زر "إعادة" لتنفيذ الإجراء الملغي
  • حفظ تلقائي: يتم حفظ البيانات تلقائياً في المتصفح
  • استعادة البيانات: يتم استعادة البيانات المحفوظة عند إعادة فتح التطبيق

اختيار نظام الإحداثيات

Application Overview

The GPS Coordinate Plotter is an advanced tool for plotting geographic coordinates and calculating areas. Specifically designed for agricultural and surveying use with an easy-to-use Arabic interface.

Interactive Mapping

  • High-resolution Google Maps
  • Multiple map types (Road, Satellite, Hybrid, Terrain)
  • Real-time coordinate display
  • Automatic coordinate system conversion
  • Support multiple coordinate systems (UTM and Lat/Lon)

Multiple Polygon Management

  • Draw multiple polygons at the same time
  • Separate management for each polygon
  • Different colors for each area
  • Separate calculations for each polygon
  • Export all polygons together

Advanced Calculations

  • Area calculation in dunums
  • Perimeter measurement in meters
  • Distance calculation between points
  • Instant and accurate results
  • Total areas for all polygons

Data Export

  • JSON Export: Contains all data in organized format
  • CSV Export: Organized spreadsheet file for analysis
  • KML Export: Google Earth map file
  • Multiple Export: Export all polygons in one file
  • Auto Naming: Files are automatically named with date and time

How to Use the Application

Entering Coordinates

Step 1: Choose coordinate system (UTM or Lat/Lon)
Step 2: Navigate to the "Coordinate Input" section
Step 3: Enter coordinates in the required format
Step 4: Adjust drawing options (color, opacity, owner name)
Step 5: Click "Draw Polygon" to view results
Step 6: Repeat the process to add more polygons

Drawing Lines

Step 1: Right-click at the starting point
Step 2: Right-click again at the ending point
Step 3: Distance will appear on the line automatically

Polygon Management

View Polygon: Click "View" in the polygon list
Delete Polygon: Click "Delete" to remove a specific polygon
Select Polygon: Click on the polygon on the map to select it

Coordinate Format

Supported Coordinate Systems

The application supports two coordinate systems:

UTM System

  • Format: Easting,Northing
  • Example: 123456.78,9876543.21
  • UTM Zone: 38N
  • Suitable for agricultural surveying

Lat/Lon System

  • Format: Latitude,Longitude
  • Example: 31.9539,35.9106
  • WGS84 system
  • Suitable for general use

Supported Formats

The application accepts coordinates in the following formats:

# Single point format Easting,Northing (UTM) 123456.78,9876543.21 Latitude,Longitude (Lat/Lon) 31.9539,35.9106 # Multiple points format (one point per line) 123456.78,9876543.21 234567.89,9876544.32 345678.90,9876545.43 # Multiple points with separator * 123456.78,9876543.21*234567.89,9876544.32 345678.90,9876545.43*456789.01,9876546.54

Important Notes

  • Choose the appropriate coordinate system before entering data
  • Minimum 3 points required to draw a polygon
  • Use comma (,) to separate coordinates
  • Use * to separate points on the same line
  • Multiple lines can be entered for multiple points
  • Coordinates will be automatically converted for map display

Map Controls

  • Show Coordinates: Right-click anywhere on the map

Undo and Redo

  • Undo: Use "Undo" button to undo the last action
  • Redo: Use "Redo" button to execute the undone action
  • Auto Save: Data is automatically saved in the browser
  • Data Recovery: Data saved can be recovered when the application is reopened

Coordinate System Selection

  • UTM System: Suitable for agricultural and surveying
  • Lat/Lon System: Suitable for general use and maps
  • Automatic Conversion: Coordinates are automatically converted for display

Export and Print Data

Export Data

  • JSON Export: Contains all data in organized format
  • CSV Export: Organized spreadsheet file for analysis
  • KML Export: Google Earth map file
  • Multiple Export: Export all polygons in one file
  • Auto Naming: Files are automatically named with date and time

Print Map

Polygon Management

Troubleshooting

Common Problems and Solutions

Map Not Loading

  • Check internet connection
  • Verify Google Maps API is working
  • Clear browser cache

Coordinates Not Plotting

  • Check coordinate format
  • Ensure minimum 3 points
  • Verify coordinates are in correct range

Export Not Working

  • Make sure polygon is drawn first
  • Check browser download settings
  • Verify file permissions
